# Генеральная Спецификация: Clinical Enrichment Flow

> Версия: 0.1.0 | Статус: Discovery (AS-IS)

## 1. Обзор
Система предназначена для автоматизации сбора медицинских данных у пациентов. Процесс начинается администратором в Telegram, продолжается агентом в WhatsApp и завершается пациентом в специализированном веб-интерфейсе.

## 2. User Stories

### Администратор клиники (Telegram)
*   **Search**: «Найди записи по имени Елена» -> Получение списка активных сессий.
*   **Enrichment Trigger**: «Отправь анкету Елене» -> Генерация ссылки и автоматическая отправка.

### Пациент (WhatsApp / Web)
*   **Receiving**: Получение персональной ссылки с приветствием.
*   **Form Filling**: Прямой ввод данных (город, жалобы, возраст) в адаптивном интерфейсе.
*   **Sync**: Мгновенное обновление профиля в CRM клиники после отправки формы.

## 3. Функциональные требования

### Агент (linda-agent)
- [x] Инструмент `find_client` для поиска по `full_name`.
- [x] Инструмент `send_enrichment_link` с поддержкой имен и ID.
- [x] Автоматическое определение недостающих полей (`missingFields`).

### Движок (psf-engine-v2)
- [x] `POST /api/clients/[id]/enrichment-link` — генерация токенов.
- [x] Auto-registration — создание записи клиента при отсутствии.
- [x] UI анкеты обогащения (`/enrichment/[token]`).

## 4. Критерии Успеха
- Ссылка генерируется и отправляется менее чем за 3 сек после команды админа.
- Данные из веб-формы попадают в базу `psf_clinic_client_profiles` в реальном времени.
- Повторный вызов анкеты для одного и того же клиента не создает дублей в базе.
