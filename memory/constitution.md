# Конституция проекта: Linda Clinical Enrichment

> Версия: 1.0.0 | Статус: Действующая (AS-IS)

## 1. Миссия и Цель
Создание бесшовного, человечного опыта взаимодействия пациента с клиникой через AI-агентов. Агент не просто чат-бот, а активный координатор, который «дозаполняет» медицинские данные, используя как чат, так и специализированные веб-интерфейсы.

## 2. Технологический Стек
*   **Backend & API**: Next.js (App Router), Node.js.
*   **Database**: PostgreSQL (через `@neondatabase/serverless`).
*   **AI Core**: GPT-4o / GPT-5-nano (через `pi-ai` и `pi-agent-core`).
*   **Messaging**: WhatsApp (Baileys), Telegram (Long-polling).
*   **UI**: Vanilla CSS + Tailwind, Next.js Components.

## 3. Принципы разработки (Законы)
1. **Stateless Logic**: Движок (`psf-engine-v2`) является единственным источником правды для бизнес-логики. Агент (`linda-agent`) — это исполнитель (Executor), который не принимает решений «от себя», а следует `activeSkill` и `missingFields`, возвращаемым движком.
2. **Auto-registration First**: Любой запрос к системе интейка должен приводить к созданию записи в базе, если клиент не найден. Ошибки `404` при обращении к пациенту — недопустимы.
3. **Identifier Neutrality**: Агент должен поддерживать поиск по человекопонятным именам (`full_name`), автоматически разрешая их в технические UUID при вызове инструментов.
4. **Security by Design**: Все инструменты администратора требуют валидации через `FIRM_SHARED_SECRET` или `PSF_AUTH_SECRET`.

## 4. Архитектурные границы
*   **Агент** не правит базу напрямую (кроме критических случаев регистрации). Все изменения идут через `patch_client_profile`.
*   **Движок** управляет портами и окружением. По умолчанию: Engine Port 3044.
