# Скиллы и Персоны (Linda Skills)

В архитектуре Linda **Скилл (Skill)** — это набор инструкций и правил поведения, которые определяют "личность" агента и его текущую бизнес-цель. Скиллы позволяют одному и тому же агенту переключаться между ролями (например, от приветливого менеджера к специалисту по анкетированию) на лету, сохраняя контекст.

## Как это работает

1. **Источник правды**: `psf-engine-v2` возвращает `activeSkill` в объекте `ClubAgentContext` перед каждым ходом.
2. **Загрузка**: Агент (LindaClientAgent или LindaAdminAgent) ищет файл `SKILL.md` в соответствующей папке `skills/<skill_id>/`.
3. **Промпт**: Содержимое файла объединяется с базовыми инструкциями и метаданными сессии (Relationship State, Goal), формируя итоговый System Prompt для LLM.

## Каталог скиллов

### 1. `manager` (Базовая роль клиента)
Основная личность для свободного общения.
- **Цель**: Ответы на общие вопросы, удержание интереса, общая консультация.
- **Тональность**: Экспертная, теплая, ненавязчивая.
- **Когда используется**: По умолчанию, когда нет активных узкоспециализированных задач.
- **Инструменты**: 
    - `log_interest_signal`: Фиксация интереса к процедурам.
    - `escalate_to_human`: Передача диалога оператору при сложном вопросе.

### 2. `profile_enrichment` (Сбор данных / Анкетирование)
Полноценный навык для квалификации пациента и заполнения медицинской анкеты.
- **Цель**: Сбор недостающих критических и важных полей (анамнез, город, жалобы, противопоказания).
- **Инструменты**: 
    - `get_client_profile`: Проверка текущих знаний о пациенте.
    - `patch_client_profile`: Быстрое сохранение данных прямо из чата.
    - `get_missing_fields`: Список полей, которые еще нужно узнать.
    - `request_enrichment_link`: **Генерация ссылки на анкету (Premium UI)**.
- **Принцип работы**: 
    1. Агент видит список `missingFields`.
    2. Если полей мало (1-2), агент спрашивает их в чате и использует `patch_client_profile`.
    3. Если полей много или они требуют сложного ввода (например, выбор из списка противопоказаний), агент вызывает `request_enrichment_link` и предлагает пациенту заполнить анкету в красивом интерфейсе.
    4. **Важно**: После получения ссылки агент должен вежливо попросить пациента её заполнить и подождать.

### 3. `booking_consultation` (Запись на прием)
Помощь в выборе времени и процедуры.
- **Цель**: Конвертация интереса в конкретную запись.
- **Тональность**: Четкая, ориентированная на результат.
- **Инструменты**: Стандартный набор для работы с календарем (в разработке).

### 4. `admin_assistant` (Операционный помощник)
Единственный скилл для `LindaAdminAgent` (Telegram), используемый персоналом клиники.
- **Цель**: Мониторинг и управление активными сессиями клиентов.
- **Возможности**: Просмотр статусов анкет, отправка сообщений "от имени" Линды, принудительное изменение полей профиля.
- **Инструменты**: `list_sessions`, `view_session`, `send_to_client`, `override_field`.

## Руководство по созданию нового скилла

Чтобы добавить новый скилл:

1. **Создайте директорию**: `packages/linda-agent/skills/<id_скилла>/`.
2. **Создайте файл `SKILL.md`**:
    - Опишите роль ("Кто ты?").
    - Укажите правила общения ("Как ты говоришь?").
    - Опишите политику использования инструментов ("Когда и что вызывать?").
3. **Зарегистрируйте**: Добавьте ID в `LindaRuntimeConfig.clientAgent.enabledSkills` (через `config.ts` или переменные окружения).

### Советы по написанию `SKILL.md`
- **Лаконичность**: Не пишите огромные тексты, LLM лучше следует четким спискам.
- **Приоритеты**: Явно указывайте, какие инструменты важнее (например, "Сначала проверь профиль, потом спрашивай").
- **Язык**: Пишите инструкции на русском, если основная работа идет с русскоязычными пациентами (это помогает выдерживать стиль).

## Взаимодействие со ссылками на анкеты

Скилл `profile_enrichment` теперь обладает возможностью генерации "премиальных" ссылок. 
- **Когда предлагать ссылку?** Когда в `missingFields` больше 3 полей или когда вопросы требуют вдумчивых ответов.
- **Как это выглядит для пациента?** Линда присылает ссылку на `http://linda.clinic/enrichment/<token>`, где пациента ждет анимированная анкета в фирменном стиле.
- **Автоматизация**: После того как пациент заполнит анкету по ссылке, движок переключит `activeSkill` обратно на `manager`.

## Управление и Запуск (Orchestration)

Для удобства разработки создана единая система запуска всей клинической экосистемы (Engine + Linda Agent).

### Основные команды (из корня pi-mono):

- `npm run dev:linda` — **Основная команда**. Запускает одновременно Next.js Backend (на порту 3044) и Linda Agent (Telegram + WhatsApp) с синхронизированным контекстом.
- `node scripts/launch_clinical_system.mjs` — То же самое, прямой запуск скрипта оркестрации.

### Раздельный запуск:

Если требуется запустить компоненты в разных терминалах:
1. **Engine**: `npm run dev -w @products/clinic-profile-os` (в репозитории `psf-engine-v2`) — запуск на порту 3044.
2. **Agent**: `npm start` (в `packages/linda-agent`) — запустит скомпилированную версию агента.
